# dateChain — Дизайн-система (стили и анимации под Tinder)

> Цель: визуал и поведение максимально близки к Tinder. Меню и навигация — идентичны по структуре.
> Все токены живут в `packages/ui` и переиспользуются всеми фронтами.

## 1. Бренд-токены (design tokens)

### Цвета

- **Основной градиент** (Tinder Flame): линейный `#FD267A → #FF6036` (розово-оранжевый).
  Используется в лого, primary-кнопках, активных вкладках.
- Акцент Like: зелёный `#22E39C` / бирюзовый градиент.
- Акцент Nope: красный/розовый `#FD5068`.
- Super Like: голубой `#1EC8FF` (звезда).
- Boost: фиолетовый `#B04AF6`.
- Фон: белый `#FFFFFF` (light) / near-black `#0E0E0E` (dark).
- Текст: `#000000` / `#8F8F8F` (вторичный).
- Разделители: `#EBEBEB`.

### Типографика

- Шрифт: близкий к Tinder — `Proxima Nova` заменяем на бесплатный аналог
  (`Montserrat` / `Inter` / системный SF). Токен `--font-family`.
- Масштаб: 12 / 14 / 16 / 20 / 24 / 32, веса 400/600/700.

### Радиусы и тени

- Карточка профиля: `border-radius: 8px`, крупная мягкая тень.
- Круглые кнопки действий (FAB-style) — как в оригинале: белые круги с цветной иконкой и тенью.
- Модалки: скруглённые верхние углы на mobile (bottom-sheet).

### Сетка/отступы

- 4px базовая сетка (4/8/12/16/24/32).

## 2. Ключевые визуальные компоненты

- **Логотип-пламя** (SVG с градиентом).
- **SwipeCard** — карточка профиля: фото на всю площадь, градиент снизу, имя+возраст, кнопка «i».
- **ActionBar** — ряд круглых кнопок: Rewind, Nope, Super Like, Like, Boost (порядок как в Tinder).
- **MatchScreen** — «It's a Match!» с двумя аватарами и градиентным фоном.
- **Нижняя навигация (mobile)** — иконки: Discovery (пламя), Explore(флаг, скрыт в core), Likes,
  Chats, Profile. Активная вкладка окрашена градиентом.
- **Боковая навигация (desktop)** — левая колонка + центральная колода + правая панель профиля/чата
  (трёхколоночный layout как в web-версии Tinder).

## 3. Анимации (Framer Motion)

- **Свайп карты**: `drag` по X/Y, поворот пропорционально смещению (`rotate = x * 0.05`),
  прозрачные метки LIKE/NOPE/SUPER появляются по мере перетаскивания (opacity от смещения),
  порог срабатывания + «улёт» карты за экран (spring), возврат при недостаточном свайпе.
- **Появление следующей карты**: масштаб `0.95 → 1`, лёгкий подъём стопки.
- **Match-экран**: fade + scale-in аватаров, всплытие кнопок.
- **Кнопки действий**: pulse при нажатии, срабатывание синхронно с жестом.
- **Like/Super Like**: всплывающая иконка-«штамп».
- **Переходы вкладок**: crossfade/slide.
- **Typing-индикатор**: три пульсирующие точки.
- **Skeleton-загрузка**: шиммер.

Требование: 60 fps на mobile, `will-change`/GPU-трансформации, `prefers-reduced-motion` уважается.

## 4. Адаптив (breakpoints)

- `mobile`: < 768px — одна колонка, нижняя навигация, bottom-sheets, свайпы жестами.
- `tablet`: 768–1024px — колода по центру, компактная навигация.
- `desktop`: > 1024px — трёхколоночный layout, действия кнопками + drag мышью.

## 5. Storybook как источник правды

- Каждый компонент UI-kit имеет story: состояния, controls, docs.
- Interaction-тесты (play-функции) проверяют поведение (свайп, клики, валидация).
- Визуальная регрессия — снапшоты историй (Chromatic-совместимо / Playwright screenshots).
- Тема light/dark переключается тулбаром Storybook.
