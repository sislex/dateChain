# dateChain — Пользовательские интерфейсы и полный список фич

Три роли/интерфейса: **Пользователь**, **Администратор/Модератор**, **Система (backend)**.
Каждый интерфейс существует в desktop- и mobile-версии (адаптив).

---

## A. Интерфейс ПОЛЬЗОВАТЕЛЯ (`user-web`)

### A1. Онбординг и авторизация

- Экран приветствия (лого, слоган, кнопки входа).
- Регистрация/вход по телефону (OTP-код) — основной путь Tinder.
- Альтернативный вход по email + OTP.
- (Опц. флаг) вход через соц-провайдеров — задел, выключено в core.
- Приём Terms / Privacy / Cookie согласия.
- Проверка возраста (18+), ввод даты рождения.
- Многошаговый онбординг профиля:
  - Имя, дата рождения.
  - Пол и «показывать меня как».
  - Кого ищу (пол интересующих).
  - Загрузка первых фото (мин. 2).
  - Разрешение геолокации / ручной выбор города.
- Восстановление сессии (refresh-токен), выход из аккаунта.

### A2. Основной экран — свайпы (Discovery / «колода»)

- Стек карточек профилей (фото, имя, возраст, расстояние, био).
- Жесты и кнопки: **Nope** (свайп влево), **Like** (свайп вправо), **Super Like** (свайп вверх), **Rewind** (вернуть карточку — премиум-флаг), **Boost** (флаг).
- Анимация перетаскивания карты, тень, поворот, метки LIKE/NOPE поверх фото.
- Пролистывание фото внутри карточки (тап по краям), индикаторы фото.
- Разворот карточки → подробный профиль (био, интересы, работа, образование, расстояние).
- Экран «мэтч!» (двойное фото, кнопки «Написать сообщение» / «Продолжить свайпать»).
- Пустая колода (закончились кандидаты) — состояние с CTA (расширить радиус/фильтры).
- Дневной лимит лайков (core: настраиваемый лимит, экран «лимит исчерпан»).

### A3. Фильтры подбора (Discovery Settings)

- Радиус поиска (км), с картой/слайдером.
- Возрастной диапазон (слайдер).
- Пол интересующих.
- «Показывать меня в Discovery» (вкл/выкл — режим невидимости).
- Глобальный режим (флаг Passport — выключено в core).

### A4. Мэтчи и чат

- Список мэтчей (горизонтальная лента «новые мэтчи» + список диалогов).
- Индикаторы: новый мэтч, непрочитанные, «печатает…», онлайн-статус.
- Чат: текст, эмодзи, отправка изображений (из галереи), GIF (флаг).
- Статусы сообщений: отправлено / доставлено / прочитано.
- Real-time доставка (Socket.IO), typing-индикатор, presence.
- Отмэтчить (unmatch), пожаловаться на пользователя из чата.
- Пустое состояние (нет мэтчей).

### A5. Профиль пользователя (свой)

- Просмотр своей карточки как её видят другие (предпросмотр).
- Редактирование: фото (загрузка/удаление/переупорядочивание drag&drop, выбор главного), био, интересы (теги), работа, должность, образование, рост, «что ищу».
- Процент заполненности профиля.
- Верификация (базовая — флаг).

### A6. Настройки

- Аккаунт: телефон/email, изменение.
- Уведомления (push/email — тумблеры).
- Приватность: контроль отображения расстояния/возраста, режим невидимости.
- Discovery-настройки (дублируют A3).
- Язык интерфейса.
- Заблокированные контакты (block list).
- Управление подпиской (флаг монетизации — заглушка).
- Юридическое: Terms, Privacy, лицензии.
- Пауза аккаунта, удаление аккаунта (с подтверждением), выход.

### A7. Уведомления

- Центр уведомлений (новый мэтч, новое сообщение, super like получен).
- Realtime-тосты/бейджи в шапке.

### A8. Безопасность и модерация (со стороны пользователя)

- Пожаловаться на пользователя (категории: спам, оскорбления, фейк, 18-, и т.д.).
- Заблокировать пользователя.
- Safety-центр (советы безопасности — статический контент).

## B. Интерфейс АДМИНИСТРАТОРА / МОДЕРАТОРА (`admin-web`)

### B1. Авторизация и роли

- Вход по email + пароль + 2FA (TOTP).
- RBAC: `SUPER_ADMIN`, `ADMIN`, `MODERATOR`, `SUPPORT`, `ANALYST` (только чтение).
- Аудит-лог всех действий администраторов.

### B2. Дашборд

- Ключевые метрики: DAU/MAU, новые регистрации, кол-во мэтчей, сообщений, свайпов.
- Графики по времени, воронка онбординга.
- Очередь модерации (кол-во открытых жалоб).

### B3. Управление пользователями

- Список/поиск/фильтры (по статусу, дате, стране, флагам).
- Карточка пользователя: профиль, фото, устройства, история входов, мэтчи, жалобы.
- Действия: бан/разбан, заморозка, сброс фото/био, принудительная верификация,
  ручное удаление, вход «от имени» (impersonate — только SUPER_ADMIN, с аудитом).

### B4. Модерация контента

- Очередь жалоб (сортировка по приоритету/дате).
- Просмотр репортов, контекст переписки/профиля.
- Модерация фото (одобрить/отклонить/забанить), очередь авто-флагов.
- Решения: предупреждение, скрытие контента, бан, эскалация.
- Модерация текста чата по флагам (без нарушения приватности — по репортам).

### B5. Модерация профилей

- Очередь новых/изменённых профилей (флаг авто-модерации).
- Проверка фото на соответствие правилам.

### B6. Аналитика и отчёты

- Отчёты по вовлечённости, ретеншену, гео-распределению.
- Экспорт CSV.

### B7. Конфигурация системы

- Управление feature-flags.
- Настройка лимитов (лайки/день, радиус max, размеры медиа).
- Управление списками причин жалоб, шаблонами уведомлений.
- Контент Safety-центра / юридических страниц (CMS-лайт).

### B8. Поддержка (Support)

- Тикеты/обращения пользователей (базовая очередь).
- История коммуникаций.

## C. СИСТЕМА / BACKEND (`apps/api`) — доменные возможности

### C1. Auth-домен

- OTP-генерация/валидация, JWT выпуск/ротация, устройства/сессии, logout-all.
- RBAC-guard, 2FA для админов.

### C2. User/Profile-домен

- CRUD профиля, фото (порядок, главное), интересы, атрибуты.
- Процент заполнения, статусы (active/paused/banned/deleted).

### C3. Discovery-домен

- Формирование колоды кандидатов: гео (PostGIS `ST_DWithin`), фильтры (возраст/пол/радиус),
  исключение уже свайпнутых/заблокированных/отмэтченных, анти-повтор.
- Ранжирование (core: recency + расстояние; ML — флаг на будущее).

### C4. Swipe/Match-домен

- Запись лайков/нопов/суперлайков, идемпотентность.
- Детекция взаимного лайка → создание мэтча → событие в сокет.
- Дневные лимиты, rewind.

### C5. Chat-домен

- Треды по мэтчам, сообщения (текст/изображение), статусы прочтения.
- Socket.IO gateway: rooms по мэтчу, typing, presence, доставка.
- Хранение истории, пагинация.

### C6. Media-домен

- Загрузка (multipart) на локальный диск, валидация типа/размера, `sharp` ресайз + thumbnails + blur-hash.
- Защищённая отдача (проверка прав), удаление.

### C7. Notification-домен

- Внутренние уведомления, realtime-эмиты, (push/email — адаптеры за флагом).

### C8. Moderation-домен

- Жалобы, блокировки, очереди, авто-флаги (эвристики), решения, аудит.

### C9. Admin-домен

- Пользователи, метрики, конфиг, feature-flags, аудит-лог.

### C10. Инфраструктура

- Миграции, сиды (демо-пользователи/фейковые профили для дев/демо), health-checks,
  rate-limiting (Redis), логирование, конфигурация через env, OpenAPI.

---

## D. Сквозные фичи (во всех интерфейсах)

- Адаптив desktop/mobile, тёмная тема (Tinder-градиент), анимации.
- i18n-инфраструктура.
- Единый UI-kit (`@datechain/ui`) во всех фронтах.
- Обработка ошибок/лоадеры/скелетоны, offline-состояния.
- Единая система нотификаций/тостов.
